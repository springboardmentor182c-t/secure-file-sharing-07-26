# TrustShare — Group D

TrustShare is Group D's secure file-sharing application. It uses a React frontend,
a FastAPI backend, PostgreSQL metadata storage, and encrypted file storage.

## Local development

### Without Docker

Backend:

```powershell
cd server
Copy-Item .env.example .env
python -m pip install -r requirements-dev.txt
python -m src.main
```

Frontend (in another terminal):

```powershell
cd client
Copy-Item .env.example .env
npm ci
npm start
```

Open `http://localhost:3000`. The API and interactive documentation are at
`http://localhost:8000` and `http://localhost:8000/docs`.

### With Docker Compose

Run this from `server/`:

```powershell
Copy-Item .env.example .env
docker compose up --build
```

This starts React, FastAPI, PostgreSQL, and persistent local volumes.

## Production architecture

The production Docker image builds React and serves the compiled application from
FastAPI. The API base URL is supplied explicitly through `REACT_APP_API_URL` at
build time, even though the frontend and API share one HTTPS origin. PostgreSQL
stores application metadata and encrypted file payloads. Per-file AES keys are
wrapped by a separate production master key before they are stored in the database.

The repository-level `render.yaml` defines:

- `trustshare-group-d`: Docker web service in Singapore
- `trustshare-group-d-db`: PostgreSQL database
- generated production `SECRET_KEY` and separate `MASTER_KEY`
- public frontend, backend, API, and CORS URLs derived from Render's generated
  `RENDER_EXTERNAL_URL`
- `/health` deployment health checks

## Deploy to Render

1. Merge or push the deployment branch to GitHub.
2. In Render, choose **New > Blueprint** and connect this repository.
3. Select `render.yaml` and review the services before applying it.
4. After the first successful deploy, open the service URL and check `/health`.
5. Add optional SMTP and OAuth secrets in the Render dashboard if those features
   are part of the demonstration.

The Blueprint uses Render's free web-service and PostgreSQL plans. The free web
service sleeps when idle, and the free database has Render's current retention and
storage limits. Check the Render dashboard before applying the Blueprint to confirm
that both resources still show the free plan. This setup is intended for the
internship demonstration rather than permanent production use.

## Production environment variables

| Variable | Required | Purpose |
| --- | --- | --- |
| `DATABASE_URL` | Yes | PostgreSQL connection URL; supplied by Render |
| `SECRET_KEY` | Yes | JWT signing key; generated by Render |
| `MASTER_KEY` | Yes | Wraps per-file AES keys; generated by Render |
| `ENVIRONMENT` | Yes | Must be `production` in production |
| `DATA_DIR` | No | Filesystem root for optional local storage utilities |
| `REACT_APP_API_URL` | Yes | Public API base URL embedded into the React build |
| `FRONTEND_URL` | Yes | Public app URL used in links and redirects |
| `BACKEND_URL` | Yes | Public API URL used for OAuth callbacks |
| `BACKEND_CORS_ORIGINS` | Yes | Comma-separated allowed frontend origins |
| `SMTP_*` | No | Email delivery credentials |
| `GOOGLE_*` | No | Google OAuth credentials and callback URL |
| `MICROSOFT_*` | No | Microsoft OAuth credentials and callback URL |

Never commit `.env`, database files, uploaded files, or encryption keys.

For local development, these URLs come from the copied `.env` files. On Render,
the Blueprint self-references the service's generated `RENDER_EXTERNAL_URL` and
passes it into all four variables. Render exposes `REACT_APP_API_URL` to the
Docker build as a non-secret build argument.

## Deployment smoke test

After deployment, verify:

1. `/health` returns `status: ok`.
2. Signup, login, token refresh, and logout work.
3. Dashboard, Shared With Me, Analytics, Activity, and Notifications load.
4. A file can be uploaded, downloaded, shared, and opened by the recipient.
5. The uploaded file still works after a service restart.
6. Directly refreshing `/dashboard` or another React route does not return 404.
7. OAuth and password-reset links use the public HTTPS URL when configured.
